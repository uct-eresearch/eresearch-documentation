# Research data storage

## What this service is

UCT provides a Research Data Store (RDS) for secure, high-performance storage of research data.

The RDS is an on-premise storage system managed by ICTS and hosted in the UCT data centre. It is designed to support data-intensive research by providing:

- reliable, scalable storage
- fast access to data
- integration with HPC
- secure, controlled access for collaborators

---

## Who this is for

RDS is suitable for researchers who:

- work with large or growing datasets
- need reliable, long-term access to active data
- require secure storage for sensitive or regulated data
- need to connect data to HPC for analysis
- collaborate with others on shared datasets

---

## What RDS provides

### Secure, on-premise storage
- data is stored in UCT’s data centre
- access is controlled via UCT credentials
- supports compliance with South African data regulations (e.g. POPIA)

---

### High performance
- fast input/output (I/O)
- low-latency access from campus and HPC
- suitable for data-intensive workflows

---

### Scalable storage
- storage can be increased as your project grows
- no need to migrate data when expanding capacity

---

### Reliable access and backup
- data is backed up regularly
- replicated to an off-site location for disaster recovery
- designed to reduce risk of data loss

---

### Collaboration support
- access is controlled by the data owner (PI)
- supports shared access within research groups
- external collaborators can be granted access (via UCT accounts)

---

### HPC integration
- RDS can be mounted on the HPC cluster
- data can be accessed directly during computation
- avoids duplication of large datasets

---

## How RDS works

RDS storage is provisioned per project:

- storage is allocated based on your requirements
- access permissions are defined by the data owner
- data can be accessed:
  - as a network drive on your computer
  - via VPN from outside campus
  - from HPC compute nodes

---

## Cost model

RDS operates on a **cost-recovery model**:

- pricing is based on storage size (per GB/TB per month)
- there are no hidden costs for:
  - uploading data
  - downloading data
  - data transfer within UCT systems

This allows researchers to:
- budget accurately in advance
- avoid unexpected charges associated with some cloud services

---

## What RDS supports

RDS is designed for:

- active research data
- large datasets
- collaborative projects
- HPC-integrated workflows
- data requiring controlled access

---

## What RDS does not replace

It is important to choose the right tool for the job.

RDS is **not** intended for:

- long-term archival storage of inactive data  
  → use tape storage or DIRISA instead

- lightweight document collaboration  
  → use OneDrive / SharePoint

- data sharing interfaces  
  → use Nextcloud to share or transfer data

---

## How RDS fits with other services

RDS is part of a broader research ecosystem at UCT:

- **HPC (High Performance Computing)**  
  → run analyses on data stored in RDS  

- **Nextcloud**  
  → share and manage RDS data through a web interface  

- **Data transfer tools (e.g. Globus)**  
  → move data to and from RDS  

- **GitLab**  
  → manage code that operates on data stored in RDS  

---

## When to use RDS

Use RDS when:

- your data is large or growing
- you need reliable, persistent storage
- your workflow involves HPC
- your data must remain in South Africa
- you need controlled access for collaborators
