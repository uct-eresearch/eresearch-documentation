# High Performance Computing (HPC)

Run large-scale and compute-intensive research workloads on shared infrastructure.

## What this service is

UCT High Performance Computing (HPC) provides shared computational infrastructure for research that cannot be performed efficiently on a laptop or standard workstation.

It enables researchers to run large-scale, compute-intensive, and data-intensive workloads using centrally managed resources.

---

## When to use HPC

Use HPC when your work:

- takes hours or days to run on a local machine  
- involves large datasets or high memory requirements  
- requires parallel or distributed computation  
- needs access to GPUs  
- must run reliably as batch jobs  

HPC is typically used when moving from exploratory work to production-scale execution.

If your work is lightweight or short-running, another service may be more appropriate.

→ See: [Choose the right service](../../start-here/choose-the-right-service.md)

---

## What HPC provides

HPC provides:

- access to shared CPU and GPU compute resources  
- scheduled batch execution via a job scheduler  
- interactive and browser-based access options  
- integration with high-performance research data storage  
- managed software environments for scientific computing  

Workloads are executed on compute nodes via scheduled job submission rather than direct execution.

---

## Access

To use HPC, you must have an approved HPC account.

Access requires:

- an active UCT staff or student account  
- supervisor (PI) approval  

→ See: [Apply for an HPC account](../../how-to/hpc/apply-for-hpc-account.md)

---

## How HPC fits with other services

HPC is typically used as part of a broader research workflow.

It is commonly used with:

- [Research data storage](../storage/index.md) to store and manage active datasets  
- [Data transfer](../data-transfer/index.md) to move data between systems and collaborators  
- [Research software](../research-software/index.md) to develop and manage code and workflows
