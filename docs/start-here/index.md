# Start here

## What this documentation helps you do

Research at UCT often involves a combination of data, code, storage, transfer, and compute.

This documentation helps you:
- understand how these pieces fit together
- choose the right tools and services for your work
- take the next step with confidence

---

## How to navigate this site

There are three main ways to find what you need.

### 1. Understand your research workflow

If you want to see how different stages of research connect — from planning to analysis to sharing results — start here:

→ [Research workflow at UCT](../index.md/#research-workflow-at-uct)

This gives you a high-level view of how tools and services fit together across a typical research project.

---

### 2. Choose the right service

If you are unsure which platform or tool to use — for example, where to store your data or how to share it — use the decision guide:

→ [Choose the right service](choose-the-right-service.md)

This helps you compare options and avoid common mistakes (such as using collaboration tools as storage, or choosing the wrong system for large datasets).

---

### 3. Go directly to what you need to do

If you already know your goal, go straight to:

→ Tasks

This section is organised by common research activities, such as:
- running large-scale analysis
- storing and sharing data
- transferring data securely
- collaborating on code

---

## A note on how this site is organised

- **Tasks** help you get things done  
- **Services** explain the tools available  
- **How-to guides** show you step-by-step what to do  
- **Good practice** provides guidance that applies across tools  
- **Reference** contains technical lookup information  

You do not need to read everything — use the paths above to go directly to what you need.
